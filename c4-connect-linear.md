# Connect Linear

Use this page when Linear should create documentation work in Doc Holiday. Linear issues and issue comments can start new work requests, so this is the setup to use when those events should trigger docs changes.

## Set up the Linear Source

1. Enter **Source Name**.
2. Enter **API Key**.
3. Choose **Team**. Linear teams back the Team picker. When nothing is selected, it shows **Select team**. While Doc Holiday loads teams, it shows **Loading...**. Use **Search teams...** to filter the list. If no teams match, it shows **No teams found**.

## What it enables

- New Linear issues can trigger documentation work.
- New issue comments can trigger documentation work.
- In issue comments, `@doc.holiday` mentions behave like mentions in git comments, so a comment that starts with the mention can request work in the same way.

See [Request work from GitHub, GitLab, or Bitbucket with @doc.holiday](/f1-request-work-in-git.md) for the request pattern, and [Configure triggers: when Doc Holiday reacts on its own](/d2-configure-triggers.md) for trigger settings.

## Verify

Open **Sources** and confirm the Linear Source shows **Healthy**.
