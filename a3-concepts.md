# Concepts

This page defines the shared vocabulary used throughout Doc Holiday’s documentation. Read it first when the terms around Organizations, Publications, Sources, the Library, and Work History start to blur together, or when a draft needs the model put back in order.

## Organization

An Organization is the highest level home for everything Doc Holiday keeps together, from the work it writes to the guidance that shapes it day after day. It gathers every Publication, every Source, and the Library that holds shared guidance. That gives one team a single place to manage the material that shapes each request, the shared expectations behind it, and the scope each request belongs to when more than one team touches the same body of work. It also sets the boundary for the rest of the model, so every other concept on this page belongs inside it and inherits the same shared ownership. Because Organizations sit at the top, they decide which Publications, Sources, and Library entries share the same workspace.

## Publication

A Publication is the documentation destination Doc Holiday keeps current, and it gives every request a clear place to land. It is the place where work lands after Doc Holiday reads the relevant Sources and prepares a reviewable result. A Publication can point at more than one Source, which lets one destination gather context from several places while still keeping one focus and one set of expectations. It gives the work its purpose by showing which documentation set should change, which pages deserve attention first, and what the reviewer should expect to see before the result moves forward. A Publication sits under one Organization and depends on Sources for the facts it needs.

## Sources

Sources are the connected systems Doc Holiday uses when it works on a Publication, whether the Source holds the material to update or the context that explains why the update matters. One Source can provide the place where Doc Holiday writes, while another can supply context that only needs to be read. Some Sources are repositories, while others are context systems such as Notion, Confluence, Jira, Linear, or Zendesk, and each one contributes a different kind of evidence about the request. That mix lets a Publication gather the right facts without forcing every connection into the same shape or asking every team to work in the same tool; see [Sources](/c1-sources.md) for a deeper look at how these roles fit together. Because a Publication points at Sources rather than owning the information itself, the Source list shows where the facts for a request come from and which systems helped shape the final page.

## The Library and instructions

The Library is the organization wide home for reusable writing guidance, and it gives the whole Organization a steady voice. Each instruction in the Library captures a piece of guidance that Doc Holiday can apply broadly or limit to specific Publications. That lets one Organization keep a shared voice while still tuning individual Publications when they need different wording, structure, scope, or tone for different kinds of documentation. An instruction stays separate from the Publication it serves, so the same guidance can travel with the Organization and still fit more than one destination without drifting apart. See [the Library](/e1-the-library.md) for how the shared guidance is organized and applied.

## Work History

Work History is the record of one request from the moment it starts until it reaches a result, so it gives each request a clear trail. It shows what came in, what Doc Holiday did, how the request moved forward, and what changed along the way as the work progressed. The lifecycle is simple: it opens, does its work, and settles into a result such as awaiting review, merged, or closed. That makes Work History the place to look when a team wants to follow one request from start to finish, compare one pass with the next, or confirm which result came out of the work before it settles into the record. See [Work History](/f4-work-history.md) for the full page.

Taken together, these terms describe one path from source to page. Something happens in a Source, Doc Holiday turns it into reviewed documentation in a Publication, the Library keeps the guidance consistent, and Work History records the result.
