# Request work from GitHub, GitLab, Bitbucket, or Linear

Mention `@doc.holiday` in the thread you are already using, and Doc Holiday starts the request there. Put `@doc.holiday` first or within the first few words, then say what you want updated.

## Where it works

- GitHub issue comments and pull request comments.
- GitHub pull request review bodies and pull request review comments.
- New GitHub issue bodies when the request starts the issue.
- GitLab issue notes and merge request notes.
- Bitbucket pull request comments.
- Linear issue comments. If Linear is not connected yet, see [Connect Linear](/c4-connect-linear.md).

## What happens next

Doc Holiday replies in the same thread and keeps the response with the request. If more comments come in on the same PR or issue, they stay with the existing Work History entry instead of starting a separate one.

## Help shortcut

Use `@doc.holiday help` when a short reminder is enough. Doc Holiday returns inline usage help and examples.
